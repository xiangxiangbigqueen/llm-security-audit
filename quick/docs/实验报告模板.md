### 3.4 多 Agent 协同架构

本系统采用多 Agent 协同架构，利用 Claude Code 的 Agent 能力：

1. **Scanner Agent**: 负责第一层规则匹配 + 第二层 LLM 深度分析
2. **Verifier Agent**: 对扫描结果进行多维度验证确认
3. **Reporter Agent**: 汇总结果生成结构化报告

Agent 间通过 MCP (Model Context Protocol) 进行工具调用和数据交换。

## 四、关键技术

### 4.1 LLM Agent 技术
- 使用 Claude Code 的 Agent 模式进行代码语义分析
- 通过 Skill 定义封装安全审计能力
- MCP 协议集成外部工具（Git、文件系统等）

### 4.2 静态应用安全测试（SAST）
- 基于正则表达式的模式匹配（覆盖 8 类常见漏洞）
- 代码上下文语义分析减少误报
- 支持多语言（Python/JS/Java/PHP/Go 等）

### 4.3 漏洞验证技术
- 上下文分析：检查输入源、防护措施、数据流
- LLM 语义验证：利用大模型理解代码意图
- PoC 自动生成：为确认漏洞生成可执行的验证代码

### 4.4 报告自动生成
- 多格式输出（Markdown / JSON / HTML）
- 严重程度分级（高危/中危/低危/信息）
- 包含漏洞详情、代码上下文、修复建议
